PHPIDS Version 0.2 Is Out

Jun 11th, 2007

Something that started as an internal development here at Ormigo is now out. It’s called PHPIDS and I have written about it before. Congratulations to the community that has built around it and especially to .mario, who started the project here after a few days off-site brainstorming about our future.

Good to see the system get some traction and being used by some other web sites. Looking forward to see it develop.

A second internal project, this time the driving force is Timo, is Jamal. This has really helped us from the start to keep ourselves sane and to not mix html, javascript and css. Timo just recently added a mailing list to the system. I’ll just borrow from the web site to explain shortly what it is:

<em><span class="searchword0">Jamal</span> is a set of conventions and small javascript libraries to archieve a complete separation of html, css and javascript in your web application. <span class="searchword0">Jamal</span> is built on class="ext-link"><span class="icon">jQuery</span> and inspired by MVC frameworks like class="ext-link"><span class="icon">Ruby on Rails</span>, class="ext-link"><span class="icon">CakePHP</span> and its derivatives. </em>

I am really happy to see such tools spring from internal developers, giving back to the community that allows us to do what we do.

Update: I wrote PHPIDS Version 1 previously, but you should never underestimate the versioning by developers and hence, Jamal wins with 0.3, against PHPIDS 0.2 ;)
